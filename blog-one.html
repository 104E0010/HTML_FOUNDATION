  1. What is Cascading Style Sheet or CSS?

    2. Cascading Style Sheets (CSS) is a style sheet language used for specifying the presentation and styling of a document written in a markup language such as HTML or XML (including XML dialects such as SVG, MathML or XHTML). CSS is a cornerstone technology of the World Wide Web, alongside HTML and JavaScript.

    CSS is designed to enable the separation of content and presentation, including layout, colors, and fonts. This separation can improve content accessibility;[further explanation needed] provide more flexibility and control in the specification of presentation characteristics; enable multiple web pages to share formatting by specifying the relevant CSS in a separate .css file, which reduces complexity and repetition in the structural content; and enable the .css file to be cached to improve the page load speed between the pages that share the file and its formatting.

    The CSS specifications are maintained by the World Wide Web Consortium (W3C). Internet media type (MIME type) text/css is registered for use with CSS by RFC 2318 (March 1998). The W3C operates a free CSS validation service for CSS documents.

  2. Ways to Apply CSS in Web Development

    3. Type Purpose Usage
    Inline CSS Apply styles directly to HTML elements using the style attribute. Use inline CSS for quick, one-off styles or for testing purposes
    Internal CSS Define styles within the style tag inside the head section of your HTML document. Use internal CSS for styling a single document.
    External CSS Link to an external CSS file using the link tag. Use external CSS for styling multiple pages or an entire website

  3. CSS Selectors (Targeting Elements)

    4. CSS selectors define which HTML elements to style.

    1. Basic Selectors

      2. Selector Targets
      * All elements
      h1 Specific element
      .class Elements with a specific class
      #id Elements with a specific ID

    2. Advanced Selectors

      3. Selector Description
      div > p Targets p directly inside div
      div p Targets all p inside div
      p:first-child Targets the first p inside a parent
      p:hover Applies styles when hovering over p

  4. CSS Box Model (Understanding Layout)

    5. The CSS Box Model is a fundamental concept for understanding layout and design in web development. It describes how elements are structured and how their dimensions are calculated.

    1. Components of the Box Model

      1. Content
        • Description: The actual content of the element, such as text or images.
        • Example: The text inside a div.
      2. Padding
        • Description: The space between the content and the border. Padding adds space inside the element.
        • Example: Padding can be set using properties like padding-top, padding-right, padding-bottom, and padding-left.
      3. Border
        • Description: The line surrounding the padding and content. Borders can be styled with properties like border-width, border-style, and border-color.
      4. Margin
        • Description: The space outside the border. Margins create space between elements.
        • Example: Margins can be set using properties like margin-top, margin-right, margin-bottom, and margin-left.

  5. CSS Layout Techniques

    6. CSS provides various techniques to control the positioning and layout of elements.

    1. Display Property

      2. Defines how elements behave in layout.
      Value Description
      block Starts on a new line, takes full width (e.g., div)
      inline Stays on the same line, takes content width (e.g., span)
      inline-block Like inline, but allows setting width & height
      none Hides the element

    2. Positioning

      3. Controls how elements are positioned in a document.
      Position Description
      static Default (normal document flow)
      relative Moves relative to its normal position
      absolute Removed from document flow, positioned relative to nearest ancestor
      fixed Stays in place even when scrolling
      sticky Sticks to a position while scrolling

  6. Responsive Design (Mobile-Friendly Websites)

    7. Responsive design ensures websites work well on all screen sizes.

    1. Fluid Grids

      1. Purpose: Use percentage-based widths instead of fixed widths to ensure elements resize proportionally.
      2. Usage: Apply percentage values to layout elements.

    2. Flexible Images

      1. Purpose: Ensure images scale with the layout.
      2. Usage: Use CSS to make images responsive.

    3. Media Queries

      1. Purpose: Apply different styles based on the device's characteristics, such as screen width.
      2. Usage: Use media queries to create breakpoints for different screen sizes.

    4. Responsive Typography

      1. Purpose: Adjust font sizes based on screen size for better readability.
      2. Usage: Use relative units like em or rem, and media queries.

    5. Mobile-First Approach

      1. Purpose: Design for mobile devices first, then progressively enhance for larger screens.
      2. Usage: Start with styles for small screens and add media queries for larger screens.

    6. Viewport Meta Tag

      1. Purpose: Control the layout on mobile browsers.
      2. Usage: Include the viewport meta tag in the HTML .

    7. CSS Frameworks

      1. Purpose: Use frameworks like Bootstrap or Foundation that have built-in responsive design features.
      2. Usage: Utilize grid systems and responsive components provided by the frameworks.
